# Vaxil Reference Boundary

The existing Vaxil repository at `D:\Vaxil` is only a cautionary research reference.

Allowed use:

- Study mistakes.
- Compare architecture risks.
- Learn what to avoid.

Lessons to carry forward:

- Avoid god files.
- Avoid hidden routing authority.
- Avoid mixed responsibilities.
- Avoid patch stacking.
- Avoid overpatching one central controller.
- Avoid feature additions before contracts are stable.

Forbidden use:

- Do not copy Vaxil code.
- Do not import Vaxil code.
- Do not reuse Vaxil modules.
- Do not port Vaxil classes.
- Do not treat Vaxil as the Marvex architecture source.

Marvex is a new design. Vaxil is evidence of what not to repeat.
